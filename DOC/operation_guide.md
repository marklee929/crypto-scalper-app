# Heart Beat Coin Scalper – Operation Guide

## 1. 기본 운영 원칙
- 실거래 전 **최소 1개월 paper trading**
- 하루 손실 한도 / 연속 손실 제한 필수
- 변동성 급변 구간은 “아무것도 안 함”이 정답

---

## 2. Paper Trading 모드
- 실시간 가격 기반
- 주문은 가상
- 모든 이벤트는 텍스트 로그로 기록

### 로그 종류
- trades.log : 매수/매도 이벤트
- hourly_report.log : 매시간 정각 리포트

---

## 3. 리포트 항목
- 현재가
- 포지션 수량/평균단가
- 평가손익
- 실현손익
- 수수료/슬리피지
- 순손익
- 총자산(equity)

---

## 4. 실거래 전 체크리스트
- [ ] effective_gap 계산 확인
- [ ] arm_pct 적용 여부
- [ ] 변동성 필터 작동 확인
- [ ] 연속 손실 제한 설정
- [ ] API 키 권한 최소화

---

## 5. 운영 중 금지 사항
- 급변 구간에서 시장가 진입
- Oracle 신호 무시하고 강제 진입
- 손실 후 즉시 복수매매
- 로그/리포트 없이 전략 수정

---

## 6. 서버 운영 권장
- S20 이상 안드로이드 or Linux 서버
- 무인 운영 기준
- 수동 입력 최소화
